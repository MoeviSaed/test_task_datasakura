# test_task_datasakura


## Архитектура
Проект построен с использованием подхода **Feature-first** и паттерна **Composition Root**.  
Каждая фича имеет свой `Root`, который собирает зависимости и предоставляет публичный интерфейс.  
Примеры фич:  
- `AnimalWorldRoot` - отвечает за спавн животных и управление их поведением.  
- `MovementStrategy` - стратегия движения животных через `Rigidbody`.  
- `PoolContainer` - универсальный пул объектов без `MonoBehaviour`.  

В коде есть комментарии `// TODO`, где можно улучшить реализацию, оптимизировать или сделать более гибкую архитектуру.  
Я оставил их, чтобы показать намерение и подход к архитектуре, но не стал тратить на это много времени, так как это тестовое задание.  

## Особенности
- Поддержка разных типов животных: `Prey` и `Predator`.  
- Стратегия движения через интерфейс `IMovementStrategy`.  
- Пул объектов для переиспользования животных.  
- Простая система "цепи питания", которая может быть расширена.

  Готов обсудить архитектуру и возможные улучшения на TI.
